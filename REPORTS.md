# Reports — Mini Hospital v2

This document covers every report the system should produce, how to implement them consistently, and how to wire up PDF downloads via DomPDF.

---

## 1. Report Architecture Overview

Every report in this system follows the same four-layer pattern:

```
HTTP Request
    └─ ReportController         ← validates filters, returns HTML or PDF
         └─ Report Action       ← builds and returns a typed data payload
              └─ Blade View     ← renders the printable layout (HTML + PDF)
                   └─ React Page ← filter form + table preview with download button
```

### Why this separation?

- **Actions** own the query logic. They can be called from a controller, a scheduled job, an API, or a test.
- **Controllers** are thin: validate → call action → return Inertia response or PDF stream.
- **Blade views** are used only for PDF rendering via DomPDF. The React page shows the same data in a browser table.
- **React pages** handle filtering, pagination previews, and the "Download PDF" button.

## 3. Report Implementation Template

For every report, copy this pattern. Example: **Daily Revenue Report**.

### 3.1 — Action (data)

```php
// app/Actions/Reports/GenerateDailyRevenueReport.php
// php artisan make:action "Reports/GenerateDailyRevenueReport" --no-interaction

<?php

declare(strict_types=1);

namespace App\Actions\Reports;

use App\Models\Payment;
use Illuminate\Support\Carbon;

final readonly class GenerateDailyRevenueReport
{
    public function handle(Carbon $date, ?int $branchId = null): array
    {
        $query = Payment::query()
            ->with(['visit.patient', 'visit.branch'])
            ->whereDate('created_at', $date)
            ->when($branchId, fn ($q) => $q->whereHas('visit', fn ($q) => $q->where('branch_id', $branchId)));

        $payments = $query->get();

        return [
            'date'          => $date->format('d M Y'),
            'total_amount'  => $payments->sum('amount'),
            'total_count'   => $payments->count(),
            'by_method'     => $payments->groupBy('method')->map(fn ($g) => $g->sum('amount')),
            'rows'          => $payments,
        ];
    }
}
```

**Rules:**
- Return a plain PHP array — no view logic, no formatting.
- Accept typed Carbon dates and nullable foreign keys as filters.
- Eager-load what the view needs.

### 3.2 — Controller

```php
// app/Http/Controllers/Reports/DailyRevenueReportController.php
// php artisan make:controller Reports/DailyRevenueReportController --no-interaction

<?php

declare(strict_types=1);

namespace App\Http\Controllers\Reports;

use App\Actions\Reports\GenerateDailyRevenueReport;
use App\Http\Controllers\Controller;
use App\Models\Tenant;
use Barryvdh\DomPDF\Facade\Pdf;
use Illuminate\Http\Request;
use Inertia\Inertia;
use Inertia\Response;

final class DailyRevenueReportController extends Controller
{
    public function __construct(private readonly GenerateDailyRevenueReport $report) {}

    public function index(Request $request): Response
    {
        $filters = $request->validate([
            'date'      => ['nullable', 'date'],
            'branch_id' => ['nullable', 'integer', 'exists:facility_branches,id'],
        ]);

        $date   = isset($filters['date']) ? \Carbon\Carbon::parse($filters['date']) : now();
        $report = $this->report->handle($date, $filters['branch_id'] ?? null);

        return Inertia::render('reports/daily-revenue', [
            'report'  => $report,
            'filters' => $filters,
        ]);
    }

    public function download(Request $request): \Symfony\Component\HttpFoundation\Response
    {
        $filters = $request->validate([
            'date'      => ['nullable', 'date'],
            'branch_id' => ['nullable', 'integer', 'exists:facility_branches,id'],
        ]);

        $date   = isset($filters['date']) ? \Carbon\Carbon::parse($filters['date']) : now();
        $data   = $this->report->handle($date, $filters['branch_id'] ?? null);

        $tenant = Tenant::current(); // adjust to your tenant resolution

        $pdf = Pdf::loadView('reports.daily-revenue', array_merge($data, [
            'facilityName'   => $tenant->name,
            'reportTitle'    => 'Daily Revenue Report',
            'generatedBy'    => $request->user()->name,
            'appliedFilters' => [
                'Date'   => $date->format('d M Y'),
                'Branch' => $filters['branch_id'] ?? 'All Branches',
            ],
        ]))->setPaper('a4');

        return $pdf->download("daily-revenue-{$date->format('Y-m-d')}.pdf");
    }
}
```

## 4. DomPDF Tips

### Paper sizes

```php
$pdf->setPaper('a4');                        // portrait A4 (default)
$pdf->setPaper('a4', 'landscape');           // landscape for wide tables
$pdf->setPaper([0, 0, 226.77, 283.46]);      // 80mm receipt paper (POS)
```

### Stream vs download

```php
return $pdf->download('filename.pdf');  // forces browser file download
return $pdf->stream('filename.pdf');    // opens inline in browser tab (good for preview)
```

### Long table page breaks

Add this class in your Blade layout `<style>` block for tables that span multiple pages:

```css
tr { page-break-inside: avoid; }
thead { display: table-header-group; } /* repeat header on every page */
```

### Images / logos in PDFs

Use an absolute path, not a URL:

```blade
<img src="{{ public_path('images/logo.png') }}" height="40" />
```

### DomPDF config

Publish and tune `config/dompdf.php` for your environment:

```php
'options' => [
    'defaultFont'           => 'DejaVu Sans',   // best unicode support
    'isRemoteEnabled'       => false,            // disable remote image fetching
    'isHtml5ParserEnabled'  => true,
    'dpi'                   => 120,
]
```

## 8. Quick Reference — Adding a New Report

```
1.  php artisan make:action "Reports/GenerateXReport" --no-interaction
2.  php artisan make:controller Reports/XReportController --no-interaction
3.  Add route: GET /reports/x and GET /reports/x/download
4.  Create resources/views/reports/x.blade.php (@extends('reports.layout'))
5.  Create resources/js/pages/reports/x.tsx (filter form + table + Download PDF button)
6.  Write a Pest feature test covering: index loads, download returns PDF
```

That's all. Every report follows these exact six steps.
